# CHECKLIST_RELEASE — barista-school.online

## 1. Git

- `git fetch origin`
- `git status --short --branch`
- Ветка не `behind`, не `diverged`.
- Нет случайных `.env`, `.DS_Store`, временных файлов в индексе.

## 2. Проверка

```bash
bash scripts/check.sh
```

Если менялся backend:

- убедиться, что SQLite backup будет создан перед деплоем;
- проверить, что новые env-переменные описаны в `.env.example`, но реальные значения не попали в Git.
- если добавлялись author-таблицы, проверить production SQLite после рестарта API.
- если менялась модерация авторов, проверить `recipe_publications`, `recipe_publication_versions`, `recipe_publication_events` и обратную совместимость старых публикаций.
- если менялся Mixology auto-author, проверить, что приватный whitelist не попал в Git и что доступ выдаётся только по `visited`.

Если менялась admin-панель:

- менять исходники `server/admin/src/*`;
- после этого обязательно собрать `server/admin/admin-panel.js`;
- деплоить именно bundle.
- проверить light/dark layout: фон занимает всю ширину Tilda, `#adm-panel` центрирован, нет белых полей по бокам в тёмной теме.

Если менялся frontend:

- проверить desktop/mobile навигацию;
- проверить доступы `drinks`, `finance`, `author`;
- проверить no-access экран и первую доступную вкладку.
- для author mode проверить, что обычный клиентский режим `Напитки` не потерял полный набор рецептов/поставщиков.

## 3. Production smoke

```bash
npm run smoke:api
```

Для теста выдачи доступа автора и Битрикс-синхронизации:

```bash
npm run smoke:api:apply
```

Smoke проверяет:

- `https://barista-school.online/api/health`;
- admin login;
- `/api/auth/me` возвращает `access`;
- `/api/admin/users` возвращает access-флаги;
- тестовый пользователь найден по телефону/email;
- `author_profiles` существует;
- `bitrix_contact_id` совпадает с ожидаемым, если включён `expect_bitrix_synced`;
- public API не отдаёт приватные поля.

## 4. Автор рецептов

Для тестового пользователя:

- включить `Автор рецептов` в admin;
- убедиться, что доступ включился без ожидания Битрикс;
- проверить `author_profiles.bitrix_contact_id`;
- открыть контакт Битрикс и проверить метку `Автор рецептов`;
- проверить, что timeline-комментарий не дублируется при каждом сохранении профиля/аватара;
- проверить профиль автора: ФИО, телефон, описание, аватар;
- проверить, что Telegram не отображается в author frontend-форме;
- проверить, что в `Рецептурах` автора пустой старт и кнопка создания рецепта;
- проверить, что новый рецепт автора по умолчанию создаётся в группе `Авторские`;
- проверить, что неполный черновик сохраняется;
- проверить, что `На витрину` блокируется без обязательных полей и показывает список пропусков;
- проверить, что после заполнения обязательных полей рецепт отправляется на проверку;
- создать тестовый авторский ингредиент, полуфабрикат и рецепт;
- обновить страницу и убедиться, что ингредиент, полуфабрикат и рецепт не пропали;
- проверить, что фото рецепта и аватар отображаются после обновления страницы;
- проверить, что карточка публикации в профиле показывает фото, статус, комментарий и чеклист ОС;
- проверить, что повторная отправка доработанного рецепта обновляет существующую публикацию/версию, а не создаёт дубль;
- проверить, что скрытые для автора поставщики не видны: `Hiwater`, `Baristaline`, `МайТаймКап`.

## 4.1. Модерация авторских рецептов

- в admin открыть вкладку `Авторы`;
- открыть публикацию через `Проверить`;
- проверить, что видны фото, состав, оборудование, процесс, подача/срок, органолептика и видео-ссылка, если она есть;
- сохранить цену и описание витрины без смены статуса;
- вернуть на доработку с `review_flags` и комментарием;
- убедиться, что автор видит блок `Что нужно поправить` в `Мой профиль`;
- опубликовать рецепт и убедиться, что `review_flags`/`review_comment` очищены;
- снять публикацию и убедиться, что она не показывается покупателям.

## 4.2. Mixology auto-author

- импортировать whitelist только из приватного yClients-отчёта;
- проверить регистрацию телефона участника со статусом `visited` / «пришёл»: пользователь активен, `access.author=true`, `access.drinks=false`, `access.finance=false`;
- проверить регистрацию участника со статусом `no_show`, `canceled`, `pending` или `confirmed`: author-доступ не выдаётся автоматически;
- проверить, что `server/data/mixology_author_access.json` не попал в Git и не выводится в отчётах.

## 5. Публичные данные

Перед публикацией витрины проверить, что public API не отдаёт:

- телефон;
- email;
- Telegram;
- yClients ID;
- внутренние ID, которые не нужны покупателю.
